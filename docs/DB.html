Cities (Города)​

НазваниеОписание

Связи между сущностями:​

  1. Users (1) ↔ (0/N) Registrations через user_id: Один пользователь может иметь несколько регистраций / или не иметь регистрации вообще.

  2. Events (1) ↔ (0/N) Registrations через event_id: На одну акцию может быть зарегистрировано много пользователей / или может никто не зарегистрироваться.

  3. Organizers (1) ↔ (1/N) Events через organizer_id: Один организатор может проводить одну или несколько акций.

  4. Events (1) ↔ (1) Event_details через event_id: Каждая акция имеет только одно описание.

  5. Registrations (1) ↔ (0/N) Notifications через user_id: Один пользователь может получать множество уведомлений / или вообще не получать уведомления (никуда не зарегистрирован).

  6. Users (1) ↔ (1) Users_types через user_id: У каждого пользователя только одна группа крови.

  7. Cities (1) ↔ (0/N) Users через city_id: В одном городе может быть много доноров, так и не быть никого.

  8. Cities (1) ↔ (0/N) Events через city_id: В одном городе может быть как много акций, так и не быть ни одной.

  9. Cities (1) ↔ (1/N) Organizers через city_id: В одном городе может как один организатор акций, так и несколько.

  10. Events (1) ↔ (1) Status_events через event_id: Каждая акция имеет только один статус (например, "идёт регистрация", "отменена", "в процессе").

  11. Registrations (1) ↔ (1) Status_registrations через registration_id: Каждая регистрация имеет только один статус (например, "ожидание", "одобрено", "отклонено").

  12. Notifications (1) ↔ (1) Notifications_types через notification_id: Каждое уведомление имеет только один тип (например, "пуш-уведомление", "письмо на e-mail", "сообщение на телефонный номер донора").
